# 测试条目集使用说明

## 概述
本测试条目集用于验证CSL引文样式的格式化效果。包含中英文文献，覆盖多种文献类型。

## 文件说明
- `test-items.json` - CSL-JSON格式测试条目，可直接导入Zotero
- `test-items.bib` - BibTeX格式测试条目，可用于多种文献管理器
- `expected-output/` - 预期输出样本文件夹

## 文献类型覆盖
测试条目包含以下文献类型：

### 学术文献
- `article-journal` - 期刊文章（中英文各1个）
- `book` - 书籍（中英文各1个）
- `chapter` - 书籍章节（中英文各1个）
- `thesis` - 学位论文（中英文各1个）
- `paper-conference` - 会议论文（中英文各1个）

### 出版物
- `report` - 报告（中英文各1个）
- `article-newspaper` - 报纸文章（中英文各1个）
- `article-magazine` - 杂志文章（中英文各1个）

### 网络资源
- `webpage` - 网页（中英文各1个）

### 其他类型
- `manuscript` - 手稿（中英文各1个）
- `patent` - 专利（中英文各1个）
- `legislation` - 立法（中英文各1个）
- `legal_case` - 法律案件（中英文各1个）

## 自动化测试（语法验证）

### 测试脚本
项目包含以下自动化测试脚本：
- `validate.js` - CSL schema验证（XML格式验证）
- `run-tests.js` - 主测试脚本

### 运行测试
```bash
# 安装依赖
npm install

# 运行完整测试
npm test

# 仅验证CSL语法
npm run validate
```

### 自动化测试的局限性
自动化测试只能验证：
- XML格式是否正确
- 是否是有效的CSL命名空间

**不能验证的业务逻辑**：
- 引文格式是否正确（如作者-年份格式）
- 中英文切换是否工作
- 标点符号是否正确
- 页码格式是否符合期刊要求
- 文献类型标识是否正确

## 手动测试（业务逻辑验证）

根据场景选择不同的测试方式：

| 方式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **Zotero直接输出** | 日常测试、快速验证 | 快速、便于逐行对比 | 不能测试正文引用 |
| **Word文档** | 重要修改后、发布前 | 看到完整排版效果 | 创建麻烦 |

---

### 方式A：Zotero直接输出文献列表（日常测试）

#### 适用场景
- 修改CSL文件后的快速验证
- 批量测试多个条目
- 与预期输出逐行对比

#### 操作步骤

**1. 导入测试条目**
1. 打开Zotero
2. 选择"文件" → "导入..."
3. 选择`test-items.json`文件
4. 等待导入完成（应导入26个条目）

**2. 生成参考文献列表**
1. 在Zotero中选择要测试的条目（可全选）
2. 右键 → "创建参考文献文献..."
3. 选择要测试的CSL样式
4. 在弹出的窗口中选择"复制到剪贴板"

**3. 对比验证**
1. 将剪贴板内容粘贴到文本编辑器
2. 打开`expected-output/`中对应的预期输出文件
3. 逐行对比差异
4. 使用下方的检查清单验证

---

### 方式B：Word文档（深度验证）

#### 适用场景
- 重要修改后的完整验证
- 发布前的最终检查
- 需要查看正文引用+参考文献配合效果

#### 前提条件
- 安装Microsoft Word
- 安装Zotero Word插件（Zotero安装时自动安装）

#### 操作步骤

**1. 创建测试文档**
1. 打开Word
2. 创建新文档
3. 输入测试内容框架（见下方模板）

**2. 插入正文引用**
1. 将光标定位到需要插入引用的位置
2. 点击Zotero工具栏的"Add/Edit Citation"
3. 选择要引用的条目
4. 输入页码（如需要）
5. 按回车确认

**3. 生成参考文献列表**
1. 将光标定位到文档末尾
2. 点击Zotero工具栏的"Add/Edit Bibliography"
3. 自动生成参考文献列表

**4. 验证内容**
- 检查正文引用格式
- 检查参考文献列表格式
- 与预期输出对比

#### Word测试模板

```
CSL样式测试文档

测试日期：____年__月__日
测试样式：__________________

一、正文引用测试

1. 单作者引用测试
   中文：此处引用中文期刊文章[^1]
   英文：此处引用英文期刊文章[^2]

2. 多作者引用测试
   中文：此处引用多作者书籍[^3]
   英文：此处引用多作者书籍[^4]

3. 机构作者引用测试
   中文：此处引用机构作者报告[^5]
   英文：此处引用机构作者报告[^6]

4. 页码引用测试
   中文：此处引用并指定页码[^7]
   英文：此处引用并指定页码[^8]

5. 多文献同时引用测试
   此处同时引用多个文献[^9][^10]

二、参考文献列表

（此处由Zotero自动生成）

三、验证检查

□ 正文引用格式正确
□ 作者姓名显示正确
□ 日期显示正确
□ 页码显示正确
□ 多作者处理正确（等/et al.）

```

## 测试检查清单

### 通用检查项
- [ ] 作者姓名格式正确（中文用`、`分隔，英文用`&`或`and`）
- [ ] 日期格式正确
- [ ] 标点符号符合期刊要求
- [ ] 页码格式正确（如1-25 vs 1–25）
- [ ] 中英文切换正常

### Author-Date样式（社会学研究、社会、社会学评论、清华社会学评论、开放时代）
- [ ] 正文引用格式：（作者 年份: 页码）
- [ ] 多作者处理：3人以上使用"等"（中文）或"et al."（英文）
- [ ] 参考文献列表按作者字母/拼音排序
- [ ] 参考文献格式：作者. 年份. 标题. 出版信息.

### Numeric样式（武汉大学学报）
- [ ] 正文引用格式：上标数字[1]或行内数字[1]
- [ ] 参考文献列表按引用顺序排列
- [ ] 文献类型标识正确：[J]期刊、[M]书籍、[D]学位论文等
- [ ] 参考文献格式符合GB/T 7714标准

## 注意事项
- 测试条目中的DOI、ISBN等为示例数据，非真实信息
- 部分文献类型（如patent、legislation）在中文学术期刊中较少使用，但为全面性仍包含在内
- 建议定期更新测试条目，以覆盖新出现的文献类型
- 自动化测试需要Node.js环境
- GitHub Actions会自动安装所需依赖
